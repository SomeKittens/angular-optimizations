Use ng-events with caution

Earlier, we looked at how the ng-[event] directives trigger the digest cycle for us. This is the core of how Angular works but it can bite us later if we're not too careful. Things like ng-mousemove will run the digest cycle for Every Single MouseMove on the given element. Running the digest cycle 300 times will slow down even the speediest app.

Use caution with any of the following directives:

ng-
   mouse[enter/leave/move/down/up/over]
   key[down/up/press]

Instead, use CSS3's pseudo-selectors to stylize. If you absolutely must add an event, this is one of the rare cases where you'd want to attach a listener directly via a custom directive, bypassing the digest cycle entirely.
