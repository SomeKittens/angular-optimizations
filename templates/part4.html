Know when to ng-show 'em and when to ng-if 'em

(docs: ng-show / ng-if)

I see a lot of confusion about ng-hide/show vs ng-if. On the surface, they do the same thing (make an element go away and come back on command). Digging in, there's a very key difference. ng-hide/show only apply a CSS class that sets display: none. The element (and all of its watchers) is kept around. On the other side, ng-if completely removes the element from the DOM. The big question when deciding which directive to use is "Will I be updating the DOM more than I'll be needed the space in my watcher budget?" If you'll be flipping the switch a lot, choose ng-show. If the element contains quite a few watchers, it may be more prudent to go with ng-if.

(A word of warning - when ng-if removes an element, all of your custom listeners go with it, so you'll need to reattach them later.)
